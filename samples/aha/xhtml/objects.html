Conditional Inclusion of Objects

AHA! 3.0 introduces a new use of the standard XHTML object tag: the conditional inclusion of (XML) files. Unlike with the headers and footers that are unconditionally included, the object tag with aha/text type. An example: 

<object name="tutorial.conditionalobject" type="aha/text">

The conditionalobject must be the name of a concept belonging to the application tutorial. Using the authoring tools you can create Boolean expressions that determine which file will be included in the page, at the place where the object tag appears in the source.

A conditionally included file (through the object tag) must be an XML fragment with one root element. It is inserted into the parse stream of the page in which it is included. The file may contain object tags itself, causing more conditionally included objects to be considered.

When an object is included, the access to its concept causes the event-condition-action rules adaptation rules associated with that concept to be triggered. This generates user model updates, that in turn influence how the remainder of the page is adapted. When these rules not only evaluate which file to include but indeed do generate user model updates, it can cause strange adaptation effects because the part of the page above the object and the part below the object are adapted to a different user model state.
