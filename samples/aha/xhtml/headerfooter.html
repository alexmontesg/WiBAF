Headers and Footers

Headers and footers are contained in XML files (type="text/xml") that are included in a page using the object tag. They must conform to the "headerfooter" DTD that is located in /aha/AHAstandard/headerfooter.dtd (where /aha is the path parameter in the AHA! configuration). The DTD mentions the tags that are allowed in headers and footers. Besides the tags you can only add plain text. Any HTML tags must be hidden inside <![CDATA[...]]> constructs. The headers and footers of this tutorial should give some idea of what to do with headers and footers. Below is a brief list of the (meaning of the) tags that can appear in headers and footers:

  • A first group of tags consists of variables. In the header shown to the end-user the variable will be replaced by its value. (These variables are actually attributes of a "personal" pseudo-concept in the user model. The possible variables are:
    • username: The name of the end-user (if this is a field present in the registration form. The variable is used as follows: 
      <variable><username/></variable>
    • loginid: The user id entered in the registration form. (Anonymous users are assigned a special unique anonymous id.)
    • university: This field is used for on-line courses offered to students from different institutes. When available this user model variable can not just be printed in the header or footer but also be used to conditionally include fragments of information that are relevant only for students of a certain university.
    • department: Like university this field can be used to distinguish between groups of users.
    • course: The name of the current application (named "course" for historical reasons).
    • coursename: In the registration form the application can be given a "title" that is displayed in the header or footer. (The tag "title" is already used in XHTML so we had to use a different name for implementation reasons.)
    • numberdone: The number of pages in this application which the user has visited.
    • numbertodo: The number of pages in this application which the user has not yet visited.
  • The second group of tags are called handlers. They create a link anchor through which the end-user can access specially generated pages.
    • logout: When the end-user clicks on this link the session is ended. The handler is used as follows: 
      <handler>
<handlername><logout/></handlername>
<linkdescription>Logout (or any other text)</linkdescription>
</handler>
    • colorsettings: This link will access a form through which the end-user can change the color scheme for links.
    • knowledgesettings: This link will access a form that lets the end-user change the "knowledge" user model attribute of all concepts that have such attribute and for which that attribute is not marked as read-only.
    • passwordsettings: This link will access a form to change the end-user's password. Anonymous users cannot have a password.
    • done: This link will access a page showing all the pages the end-user has read. Each page name is a link to that page.
    • todo: This link will access a page showing all the pages the end-user has not (yet) read. Each page name is a link to that page.

Note: The syntax for headers and footers is considered deprecated. Headers and footers will most likely be defined in different ways in future versions of AHA!.
