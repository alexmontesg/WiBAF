The Concept Editor

The Concept Editor is an authoring tool for defining concepts and event-condition-action rules. adaptation rules. We briefly describe the use of the Concept Editor here, using a series of screen dumps to guide you through the process. We assume that you are already familiar with the Graph Author which is easier to use and intended for less technical authors.

  • When you start the Concept Editor it asks for a login and password. As of AHA! 2.0 authors are identified and have separate work spaces. The manager can add new authors, either as part of the initial installation or at a later time using the configuration menu.
  • After logging in the main Concept Editor window appears. Through the "File" menu you can create a new application, open an existing one and save it.
  • Unlike with the Graph Author the Concept Editor shows concepts with their full name which includes the application name. The Graph Author hides the application name, but does generate it when you save the application.
  • The Concept Editor shows a complete list of concepts (not a hierarchy like the Graph Editor does). Each concept can be "opened up" to reveal its attributes. The list of concepts with attributes resembles a windows explorer view. A template determines which attributes are created automatically for every new concept.
  • When selecting a concept the Concept Editor shows information about the concept, including the requirement for the concept to be suitable. For the welcome concept from this tutorial we have a screendump of what would be shown.
  • One can select an attribute (or create one) and then edit the adaptation rules associated with that attribute. The access attribute screendump shows the following information (and more):
    • Type: Each attribute is of type "int", "string" or "bool".
    • Changeable: This flag indicates whether the end-user is allowed to change the value of the attribute (through a form) or not.
    • Persistent: This flag indicates whether the attribute value is used during the rule execution for one event (page access) only, or whether the value is retained, like for "knowledge" for instance.
    • System: This flag indicates whether the attribute has a special meaning to the AHA! system. The access attribute for instance has the special meaning that it temporarily becomes true when the page is accessed.
    • Generate list item: Each line in this list contains the condition of an adaptation rule. You can select a condition and then edit the rule.
  • The adaptation rule form shows that for a rule the condition can be edited, the action to be performed if the condition is true and the optional action if the condition is false. Through the "Propagating" flag you can indicate that changes made to the user model by this rule may trigger other rules.

Note that most authors will never need to use the Concept Editor. The entire concept and rule structure for this tutorial was created using only the Graph Author.
