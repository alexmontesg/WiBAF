XHTML versus HTML

AHA! 2.0 uses pages written in XHTML. Pages are validated using the XML parser. In many cases when an end-user (hopefully you as an author trying out your application) tries to load a page with a syntax error the parser will display a "friendly" error message indicating what is wrong.

For experienced HTML authors there are a few things you should know in order to write (mostly) correct XHTML pages that will work in XHTML browsers as well as older HTML browsers:

  • All tags have to be properly nested. In HTML there is automatic ending of tags, like in the following construct: 
    <ul>
<li> ... <li> ... <li> ... 
</ul>
    Where each list item is ended automatically when the next list item starts or when the whole list ends. in XHTML you have to explicitly end all tags: 
    <ul>
<li> ... </li> <li> ... </li> <li> ... </li>
</ul>
  • In HTML there are a number of tags that have no end tag, like 
    <br>
    In XHTML all tags must be ended. There are two ways to do this: 
    <br></br> or < br />
  • Older browsers ignore explicit end tags like </br> but may get confused by the abbreviated form <br/>. A solution is to add a space between the tag name and the /, as in < br />.

Unless you are creating an application for a known audience with a single known browser it is best to try all pages on several browsers, including some new ones (like Mozilla) and some old ones (like Netscape version 4).
