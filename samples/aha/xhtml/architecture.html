AHA! Adaptive Hypermedia Architecture

Main characteristics:

The AHA! system, developed in our university, is a general-purpose adaptive web server extension. It offers adaptive presentation as well as adaptive navigation support.
  • The adaptive presentation is done through the conditional inclusion of fragments or objects. This is a low level but very powerful adaptation technique.
  • Adaptive navigation support is offered by changing the colors of link anchors. This results in link hiding or link annotation depending on the choice of colors
  • AHA! can not only perform adaptation on pages that are stored on the local server but also on pages from other servers.
  • The pages can be in html, xhtml or other xml formats. In this course we will use xhtml and smil as page formats.
  • The user model in AHA! is flexible in the sense that it can contain arbitrarily many concepts and each concept can have arbitrarily many attributes. But the attributes are of simple types, so recursive structures are not possible.
  • AHA! uses event-condition-action rules to perform user model updates. The rules can optionally trigger each other. This results in a powerful rule system, with the additional danger of termination and confluence problems.
  • Defining the user model updates and adaptation through these low level rules is difficult. AHA! therefore offers a graphical authoring tool to define concept relationships like prerequisites. These relationships are translated into the ECA rules by a compiler.
  • AHA! also contains some extra functionality, like a forms editor and the ability to generate multiple-choice tests. These features only work with html-based applications.
  • We will discuss all the aspects of the AHA! system in detail, first of all because this tells you everything about how this adaptive hypermedia system works, but also because you should learn how to create an adaptive hypermedia application using the AHA! system.
