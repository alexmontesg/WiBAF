Setting up an adaptive site with AHA!

Setting up an AHA! site is easy thanks to friendly browser-based configuration interfaces.
  1. AHA! uses Java Servlets. The easiest way to use servlets is by installing the Open Source Tomcat server. This server is available from the Apache Foundation, but we have used the Java Web Services Developer Pack or JWSDP. It works with Windows and Linux alike.
  2. AHA! can be downloaded from aha.win.tue.nl. (AHA! version supporting Learning Styles can b downloaded from http://wwwis.win.tue.nl/~nstach/PhDThesis/assignment/) We use a prerelease of AHA! 3.0. You can unpack the archive in any directory, but in our examples we will sometimes use the path c:/aha.
  3. Before you start up the Tomcat server (using startup.bin or startup.sh in the bin directory) it is best to disconnect the network interface until you have configured AHA! and changed the password.
  4. In Tomcat you should create a context for AHA!.
  5. AHA! has an automatic configuration program that edits its own configuration file based on information from the AHA! context.
  6. Once the Tomcat server is restarted the configuration can really begin. You can create authors and applications and use the authoring tools.
