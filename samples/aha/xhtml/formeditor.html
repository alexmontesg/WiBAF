The Form Editor

The Form Editor enables you to create custom forms to let end-users change values of attributes of concepts in their user model. To this end the Form Editor offers the following features:

  • You can choose an application name and load the concept structure of that application. You can only create forms that let end-users modify attribute values of concepts of a single application. The concepts and attributes are shown in a ay that resembles the Windows explorer.
  • For each attribute that is marked as changeable in the concept structure a field can be created in the form. You first have to select the attribute of the concept, and then you can create the form field. A form can contain the following types of fields:
    • Input: This is used to create a textfield, intended to enter numbers or strings. A text field has a visible "Size", a "maxLength" to indicate how many characters can be typed (can be more than the "Size"), and
    • Select: This is used to create a list of possible values of which one can be selected. The width of the list is determined by "Size". The "EnumList" field is used to enter the possible values as a semicolon-separated list.
    • Option: This is used to create radio buttons (only one choice allowed) or check boxes (more than one choice allowed).
      The "Allowed" field should have a value of 1 to create radio buttons.
      The "EnumList" is the list of choices, which must be a semicolon-separated list.
    • Button: This is used to create the "submit" or "reset" buttons.
  • The FormEditor shows an editable view of the XML file it will create for the form. You can add text, xhtml code or move pieces around. There is also a "Preview" that shows more or less what the form will look like. This Preview is created using a "standard" Java Swing class. Its understanding of html and xhtml is somewhat limited. The FormEditor assumes that a "header.xml" and "footer.xml" file can be included, but you can edit that part out.
  • When saving the form an XML file is created in the directory assigned to you as author. You have to manually move the form to the directory containing the pages of your application.
